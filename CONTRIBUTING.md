# Contributing

## Workflow

1. Open or claim an issue.
2. Create a branch with focused scope.
3. Include tests or operational verification notes.
4. Open PR with change summary, risks, and rollback path.

## PR baseline

- Small, reviewable diffs
- Clear run commands for verification
- Updated docs when behavior changes

## Operations changes

For deployment/ops changes, include:

- impact scope
- migration steps
- rollback steps
- monitoring checks
